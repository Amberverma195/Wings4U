## Saved Carts – DB-Backed Persistence, Guest Merge & Phase 3 Frontend

Issue note (original gap analysis):

- [`cart_saving_in_db_issue.md` starting at line 1](../cart_saving_in_db_issue.md#L1)

Fix / completion note (all four phases):

- [`cart_saving_in_db_fix.md` starting at line 1](./cart_saving_in_db_fix.md#L1)
- [`cart_saving_in_db_fix.md` Phase 2 completion starting at line 57](./cart_saving_in_db_fix.md#L57)
- [`cart_saving_in_db_fix.md` Phase 3 completion starting at line 81](./cart_saving_in_db_fix.md#L81)

These two files are paired. The issue note documented the original four-phase plan (DB schema existed, API/FE/checkout were missing). The fix note records each phase as it shipped:

- **Phase 1** — `packages/database/prisma/schema.prisma`: `SavedCart` / `SavedCartItem` models, `SavedCartStatus` enum, unique constraints per user/guest × location × status.
- **Phase 2** — `apps/api/src/modules/saved-cart/`: types, `CartStore` abstraction, `DbCartStore` Prisma implementation, service, controller, guest-cookie helper. Endpoints: `GET/PUT/DELETE /api/v1/cart/me`, `POST /api/v1/cart/merge`.
- **Phase 3** — `apps/web/src/lib/cart.ts` (`useCartState`): hydration from `fetchSavedCart` after `session.loaded`, 400 ms debounced `putSavedCart` on mutations, merge on login (false→true via `mergeSavedCartOnLogin`), logout wipe + guest re-fetch, `cartExpiresAt` / `isGuestCart` / `isCartHydrated` exposed in return value (fixes TS2322). Guest-cart expiry banner (`apps/web/src/components/guest-cart-expiry-banner.tsx`) shown when cart expires within 24 h; mounted in `wingkings-shell.tsx`. Merge centralized in `useCartState` — no changes to `customer-auth.tsx` or `navbar.tsx`.
- **Phase 4** — `apps/web/src/app/checkout/checkout-client.tsx`: on successful checkout calls `deleteSavedCart(locationId)` (marks DB cart `CONVERTED`) then `cart.clear()`.
